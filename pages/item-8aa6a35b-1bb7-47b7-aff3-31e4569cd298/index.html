Isi dari database adalah kumpulan tabel-tabel

Pada akhir pelajaran, siswa seharusnya dapat ...

  1. Membedakan field dan record.
  2. Menulis dan menjalankan perintah SQL untuk membuat tabel dengan struktur yang benar.
  3. Mengganti nama tabel dengan perintah SQL..


Mari kita membuat tabel tentang localhost.

Tabel tersebut akan disebut wombat, dan akan memiliki bidang-bidang ini.

wombat_idadalah INTkependekan dari integer, yaitu bilangan bulat. Ini adalah kunci utama tabel. Setiap wombat akan memiliki id yang berbeda, jadi meskipun dua wombat memiliki nama yang sama, program kami dapat menggunakan id tersebut untuk membedakannya.

wombat_idakan bertambah secara otomatis. Saat kita menambahkan data baru ke dalam tabel, DBMS akan secara otomatis membuat nilai id baru untuk data tersebut. Satu hal yang tidak perlu kita pikirkan lagi.

wombat_idtidak boleh null. (Kita sudah membahas null sebelumnya.) "Tidak null" berarti kolom tersebut wajib diisi. Setiap rekaman wombat harus memiliki id.

nameadalah VARCHAR, yang berarti dapat menampung sejumlah karakter yang bervariasi. Hingga 50, untuk bidang ini. Tidak boleh null, artinya, ini adalah bidang yang wajib diisi untuk setiap rekaman.

weightadalah FLOAT. Seperti Singledalam VBA, sebuah angka, seperti 2.1. Itu bisa bernilai null, artinya tidak diperlukan. Jika bernilai null, kita tidak tahu berat wombat.

commentsadalah TEXT. Apa pun yang dapat Anda ketik, hingga 65.535 karakter. Bisa berupa null, jadi tidak diperlukan.


Oke, mulai phpMyAdmin di localhost (ingat tombol CP Admin lama), dan pilih DB wombat.

Pada tab Struktur, beri tahu phpMyAdmin bahwa Anda menginginkan tabel bernama wombats, dengan empat kolom. Klik tombol Buka di sebelah kanan.

(Itu mengingatkanku pada cuplikan salah satu film TMNT: "Ayo ninja, ayo ninja, ayo!")

Sekarang beritahu phpMyAdmin bidang apa yang Anda inginkan.

Klik tombol Simpan. Anda akan melihat struktur tabelnya.

Adela

Jadi kita punya DB yang disebut wombats, dan tabel yang disebut wombats. Apakah setiap DB memerlukan tabel dengan nama yang sama?

Tidak. Beri nama tabel apa pun yang masuk akal. Jika kita ingin menyimpan data tentang wombat, maka tabel yang bernama wombatsmasuk akal.


Ekspor DB ke server

Oke, kita punya database di localhost. Mari ekspor apa yang kita punya ke server.

Jalankan phpMyAdmin di localhost. Setelah wombatsmemilih DB, pilih tab Ekspor.

Biarkan semua pengaturan sebagaimana adanya, dan klik tombol Go (tombol Go, tombol Go, Go!). phpMyAdmin akan membuat file dengan perintah yang dibutuhkan untuk membuat ulang tabel, dan mengirimkannya ke PC Anda. Simpan file tersebut di suatu tempat.

(Jika Anda mau, buka berkasnya di Notepad dan periksa.)


Sekarang jalankan phpMyAdmin di server Anda (dari cPanel), dan pilih DB wombats. Ingat, DB tersebut akan diberi nama rosie_wombats, atau apa pun nama pengguna Linux Anda.

Klik tab Impor, klik tombol Pilih file, dan pilih file yang baru Anda simpan.

Sekarang klik Go. phpMyAdmin akan membaca berkas tersebut, dan menjalankan pernyataan SQL di dalamnya.

Lihat pada tab Struktur, dan tabel wombat akan tercantum. Lihat struktur tabel, dan Anda akan melihat kolom yang Anda buat di localhost.
