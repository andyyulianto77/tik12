Sudah tidak sabar untuk mempraktekkan apa yang sudah dipahami tentang database. Okeee. 😃

Mari kita membuat basis data yang bisa kita mainkan.

Anda akan membuat dua, satu di localhost(PC Anda), dan satu di Reclaim. Ini adalah cara yang bagus untuk bekerja. Jalankan kode Anda di PC Anda, lalu buat duplikat DB di server Anda.

Host lokal

Di XAMPP CP (panel kontrol), jalankan Apache dan MySQL. Kemudian klik tombol DB admin.

phpMyAdmin akan dimulai. Ini adalah aplikasi web yang ditulis dalam PHP, jadi dimulai di browser.

Kini, MySQL memiliki sistem keamanannya sendiri, terpisah dari OS. Saat Anda mengakses DB MySQL, Anda melakukannya sebagai pengguna MySQL. Lebih mudah untuk membuat pengguna dan DB secara bersamaan.

Etan

Jadi pengguna MySQL tidak ada hubungannya dengan info pengguna Reclaim saya?

Benar. Pengguna MySQL adalah urusan mereka sendiri. Anda menggunakannya untuk terhubung ke DB Anda dalam program PHP, tetapi hanya itu saja.

Klik pada tab akun pengguna...

...lalu tautan Tambahkan akun pengguna:

Formulir untuk membuat akun pengguna akan muncul. Isi kolom nama pengguna dan kata sandi. Centang "Buat basis data dengan nama yang sama dan berikan semua hak istimewa."

Saya membuat pengguna dengan nama wombat, dan database bernama wombatjuga.

Klik tombol Go (kanan bawah halaman). Anda akan melihat pesan konfirmasi, dan DB baru pada daftar:


Klaim kembali akun

Mari buat DB wombat di server Reclaim Anda.

Cara termudah untuk melakukannya adalah dengan cPanel. Mudah itu bagus. Ini sama seperti yang Anda lakukan dengan phpMyAdmin di locahost, hanya dengan antarmuka yang sedikit berbeda.

Buka cPanel. Cari bagian DB. Buka wizard.

Wizard pertama-tama meminta nama basis data. Saya akan menamainya wombat juga.


Nama DB sedikit berbeda di server Anda. Ada hal-hal tambahan di awal.

Etan

Mengapa demikian?

Anda satu-satunya orang yang membuat DB di PC Anda. Jika Anda membuat DB bernama wombat, hanya ada satu.

Akun Reclaim Anda berbeda. Untuk menekan biaya, Anda berbagi satu server Linux dengan orang lain. Apa yang terjadi jika beberapa dari Anda membuat DB wombat? Argh! Namanya bentrok.

Hmm. Apa yang harus dilakukan?

Setiap pengguna Reclaim memiliki akun di server Linux mereka. Anda dapat melihat nama pengguna Linux Anda di halaman cPanel utama:

Nama pengguna Anda unik. Server Anda menggunakan fakta itu untuk membuat nama DB yang unik.

Jika nama pengguna Anda adalah rosie, direktori home Anda akan menjadi /home/rosie. Anda dapat melihatnya di File Manager cPanel.


Untuk menjaga agar nama DB Anda tetap unik, cPanel menambahkan nama pengguna Linux Anda di bagian depan:

Jadi nama sebenarnya DB di server itu adalah rosie_wombat.


Adela

Bisakah saya menggunakan nama yang sama untuk DB di localhost?

Ya, Anda bisa! Kalau begitu, hasilnya akan sama saja. Bagus! Tapi Anda tidak perlu melakukannya. Saya biasanya tidak peduli.


Klik Berikutnya untuk melanjutkan ke langkah berikutnya dalam panduan. Panduan akan meminta Anda untuk membuat pengguna.

Sekali lagi, nama Linux Anda akan ditambahkan di bagian depan. Ingat, ini hanyalah pengguna MySQL, tidak digunakan untuk hal lain.

Pastikan Anda mencatat kata sandi di suatu tempat, dan klik Buat pengguna.


Sekarang tambahkan pengguna ke DB, dengan semua hak istimewa.

Klik Langkah berikutnya, dan selesai.
