Github + Stickblitz + HAX = penerbitan web gratis

Sumber Tulisan

#situs web hax#pengembang web#komponen web#panduan

Dalam tutorial ini saya akan menunjukkan kepada Anda cara Membangun dan mengelola alur kerja penerbitan web menggunakan akun gratis dan tidak menginstal alat apa pun dengan menggunakan Github, Stackblitz, dan repo haxtheweb/haxsite.

Memulai

  • Pertama, mulailah dengan membuat akun di github
  • Berikutnya akun di stackblitz

Langkah 1: Siapkan alur kerja Halaman Github

  • Sekarang navigasikan ke repo haxtheweb/haxsite
Klik Create a new repository opsi yang ditemukan di bawah Use this template tombol (gambar di bawah)

  • Selanjutnya, beri nama repositori Anda dan pilih Create repository
Create repository
  • Ini akan memakan waktu beberapa detik dan kemudian Anda memiliki salinan repo!
  • Sekarang saatnya untuk mengaktifkan kemampuan "Halaman" Github. Kliksettings
  • Di sebelah kiri bawah Code and automationklik Pagestombol
  • Klik kata yang bertuliskan "Tidak Ada" di bawah Branchdan pilih maincabang
  • Anda akan melihat master; klik "Simpan"


Tunggu beberapa menit atau lebih dan segarkan halaman dan sekarang Anda akan melihat "Situs Anda aktif di"

Anda dapat melihat situs tutorialnya di https://btopro.github.io/jubilant-spoon/ dan sekarang kami telah mengatur semuanya sehingga ketika ada perubahan yang dibuat di github, situs tersebut akan otomatis dibangun kembali dan alamat kami akan mencerminkan perubahan tersebut. Sekarang saatnya menggunakan HAX untuk mengedit file kami di cloud!

Langkah 2: Gunakan HAX untuk mengedit situs di Stackblitz

  • Buka https://stackblitz.com dan klik tombol "Buka repositori Github [beta]"

Pilih repositori Anda setelah mengotorisasi koneksi antara github dan stackblitz dan klik "Buka repositori"
  • Ia akan berpikir sebentar, melakukan beberapa instalasi, lalu Anda akan melihat jendela pecah di sebelah kanan seperti di bawah ini (jangan panik, ini normal!)
  • Klik ikon Plug di sebelah kiri untuk "Port yang digunakan" dan kemudian klik ikon "Buka di browser"
  • Akan muncul pesan yang mengatakan "Anda hampir sampai!", klik tombol "Hubungkan ke Proyek"

Langkah 3: JACKPOT - Waktunya HAX

  • Klik "Edit" pada bilah atas untuk memulai HAXing web!
  • Tambahkan blok, edit konten, kelola kerangka situs menggunakan perancang kerangka, lanjutkan! Anda memiliki salinan HAX yang berfungsi penuh yang berjalan di stackblitz

Langkah 4 - Tutorial cepat HAX

  • Setelah mengklik Edit, klik pada konten halaman untuk mengaktifkannya. Panel kanan akan menampilkan opsi konfigurasi
  • Singkatan Markdown seperti #dan >dan -dapat digunakan untuk menulis dengan cepat
  • Merlin adalah sistem penemuan perintah. Jika Anda mencoba melakukan sesuatu, mulailah mengetik kata yang Anda pikirkan dan Merlin mungkin dapat membantu Anda untuk melakukan tindakan tersebut. Merlin sangat bagus untuk memasukkan Blok di halaman dengan menulis dan mengetik /untuk memulai paragraf kosong.
Outline Designer memungkinkan Anda mengelola hierarki situs dengan cepat dan dapat ditemukan di bawah Tindakan situs -> Desainer garis besar
  • Anda dapat mengubah tema dan detail situs di bawah Tindakan situs -> Edit pengaturan

Langkah 5 - Publikasikan perubahan ke github

Sekarang Anda telah mengubah konten, menambahkan blok, mungkin beberapa halaman, dan saatnya memperbarui konten kami di github. Mudah saja!

  • Tutup tab browser yang menjalankan HAX di cloud dan kembali ke repo Stackblitz
  • Klik "Kontrol Sumber" di sebelah kiri seperti yang ditunjukkan
  • Anda akan melihat tombol "Sinkronkan Perubahan" berwarna biru dengan jumlah perubahan. Berkat HAX + Stackblitz, setiap operasi yang Anda simpan = sebuah komitmen di repo di balik layar!
  • Ini akan menanyakan Anda tentang tarik dan dorong, tekan "OK" seperti yang ditunjukkan
  • Pertama kali Anda melakukan ini, Anda akan diminta untuk mengizinkan Stackblitz untuk berkomunikasi dengan Github. Terima izin untuk koneksi ini
  • Kembali ke repo github dan Anda akan melihat proses pembuatan dimulai. Dalam satu menit atau lebih, segarkan tautan halaman github untuk melihat perubahan Anda

SELAMAT!

Sekarang Anda memiliki alur kerja penerbitan HAX yang tidak memerlukan instalasi apa pun dan tidak membayar apa pun untuk menghosting konten Anda secara daring!

Kami akan membahas pertimbangan teknis HAX di postingan mendatang, tetapi tldr-nya;

  • KONTEN situs ini disimpan dengan situs selalu di bawah direktori pages/sehingga semuanya dapat bekerja sebagai paket yang berdiri sendiri
  • Aset JavaScript/sistem yang mendukung situs ini berasal dari jaringan CDN kami yang menggunakan Magic Script . Jika Anda ingin menjalankan aset tersebut secara lokal, Anda dapat menyalin wc-registry.json dan direktori build dari CDN kami ke tingkat teratas situs Anda.
  • Anda dapat menarik situs ini ke teknologi HAX lainnya seperti Node-js, PHP, desktop, HAXiam dan implementasi lain untuk bekerja di dalamnya.
