Apa itu Struktur Data?

Struktur Data adalah cara untuk menyimpan dan mengatur data sehingga dapat digunakan secara efisien. Sesuai namanya menunjukkan dirinya yang mengatur data dalam memori. Struktur data bukanlah bahasa pemrograman seperti C, c++, java,, python, dll. Struktur data adalah sekumpulan algoritma yang dapat kita gunakan dalam bahasa pemrograman apa pun untuk menyusun data dalam memori.

Struktur data membantu memahami hubungan antara data dan memungkinkan untuk mencari dan memanipulasi data dengan lebih efisien sesuai dengan kebutuhan aplikasi yang diimplementasikan. Dengan menggunakan struktur data yang tepat, programer dapat mengoptimalkan kinerja aplikasi dan mengatasi kompleksitas data yang tinggi.

Teknik pengolahan data

a. Batch processing:

Pengolahan data dilakukan secara berkala pada sejumlah besar data dalam satu waktu.

Contoh: Penggajian karyawan pada akhir bulan, pemrosesan transaksi kartu kredit setiap hari.

Kelebihan: Efisien untuk data dalam jumlah besar, biaya operasional rendah.

Kekurangan: Data baru harus menunggu hingga batch berikutnya untuk diproses, sehingga respons terhadap perubahan data kurang cepat.

b. Real-time processing

Data diproses segera setelah data diterima, tanpa penundaan.

Contoh: Sistem perdagangan tiket, cuaca, skor permainan, sistem reservasi penerbangan.

Kelebihan: Respons cepat terhadap perubahan data, memungkinkan pengambilan keputusan yang lebih baik secara real-time.

Kekurangan: Membutuhkan sumber daya komputasi yang lebih besar.

c. Online processing

Mirip dengan real-time processing, tetapi biasanya melibatkan interaksi langsung dengan pengguna.

Contoh: Sistem perbankan online, e-commerce.

Kelebihan: Fleksibilitas tinggi, memungkinkan pengguna untuk berinteraksi dengan sistem kapan saja dan di mana saja.

Kekurangan: Rentan terhadap gangguan jaringan

D. Multiprocessing

Penggunaan beberapa prosesor atau inti prosesor untuk menjalankan tugas secara simultan.

Contoh: Rendering video, simulasi ilmiah.

Kelebihan: Meningkatkan kecepatan pemrosesan data.

Kekurangan: Membutuhkan perangkat keras yang lebih canggih

e. Time sharing

Membagi waktu penggunaan komputer antara beberapa pengguna secara simultan.

Contoh: Sistem operasi multi-user, layanan cloud computing.

Kelebihan: Memungkinkan banyak pengguna mengakses sistem secara bersamaan.

Kekurangan: Performansi sistem dapat menurun jika terlalu banyak pengguna
