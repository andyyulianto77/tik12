Pada akhir pelajaran, siswa seharusnya dapat ...

  1. Peserta didik memahami konsep dasar client-server dalam pengembangan aplikasi web.
  2. Peserta didik dapat menjelaskan peran client dan server dalam proses komunikasi data.

Web bekerja berdasarkan model klien/server. Yaitu saat satu komputer melakukan sesuatu untuk komputer lain.

Istilah klien dan server bisa membingungkan. Terkadang, orang berbicara tentang mesin fisik, seperti PC (klien) dan server Reclaim Anda.

Terkadang, orang-orang berbicara tentang perangkat lunak. Peramban web Anda, seperti Firefox, adalah klien, dan berkomunikasi dengan perangkat lunak server web, seperti Apache. Perangkat lunak klien dan server bahkan dapat berada di komputer yang sama. Itu hal yang umum, seperti yang akan kita lihat.

Kucing di lingkungan Anda

Misalnya Anda menulis program yang menampilkan kucing-kucing di lingkungan Anda, berdasarkan rumah tangga. Berikut cuplikan layarnya:

Anda dapat mencoba aplikasinya disini.

DB Kucing

Penampilan Aplikasi ini menggunakan framework front-end dan database. Aplikasi ini berfungsi di ponsel, tablet... layar dengan ukuran apa pun. Anda akan mempelajari caranya nanti, jika Anda mengikuti bagian tersebut.

Setiap orang di lingkungan Anda mendapatkan salinan program tersebut, karena setiap orang ingin tahu tentang kucing-kucing lucu. Masalahnya, datanya selalu berubah, seiring orang-orang pindah masuk dan keluar dari lingkungan tersebut, mendapatkan lebih banyak kucing lucu, dan seterusnya. Apa yang harus dilakukan, agar setiap orang memiliki akses ke informasi terkini?

Solusinya adalah dengan membagi aplikasi menjadi beberapa bagian. Letakkan data kucing di lokasi terpusat. Saat seseorang ingin tahu tentang kucing, aplikasi akan mengambil data dari sana.

Jika data berubah, hanya basis data pusat yang perlu diperbarui. Setiap orang yang menggunakan aplikasi tersebut akan mendapatkan data baru.

Mari kita uraikan lebih lanjut.

Klien

Klien adalah peramban web/browser. Ini menunjukkan UI, antarmuka pengguna.

Antarmuka pengguna dibuat dari HTML. Cobalah aplikasinya

(Kelas ini mt-4dilengkapi dengan Bootstrap.)

Jadi, UI terbuat dari HTML.

Browser:

  • Dapatkan HTML dari server.
  • Render HTML (membuat UI).
  • Tunggu sesuatu terjadi.

  • Ketika pengguna mengklik tombol:
  • Kirim rumah tangga yang dipilih pengguna ke program PHP di server.
  • Tunggu server mengirimkan kembali beberapa HTML.
  • Tampilkan HTML.


Server

Sementara itu, server perlu mengetahui apa yang harus dilakukan saat klien mengirimkannya sebuah rumah tangga.

  • Dapatkan data yang mengidentifikasi rumah tangga dari klien.
  • Carilah kucing-kucing di rumah itu.
  • Buat HTML untuk UI, yang menyertakan data cats.

Ada dua hal di server yang perlu kita ketahui.

  • Basis data (DB), dengan data tentang kucing mana yang berada di rumah tangga mana.
  • Program PHP yang akan memeriksa basis data, membuat HTML dari hasilnya, dan mengirimkan HTML kembali ke klien.

Begini situasinya.


Server web

Dalam gambar, program Anda tampak terhubung langsung ke web. Jika demikian, Anda harus menulis kode untuk mengemas data, membongkar data, mengirim dan menerima data di jaringan... Astaga! Kedengarannya sulit!

Sulit . Jangan lakukan itu. Biarlah server web yang menanganinya untuk kita.

Server web tidak memerlukan banyak perhatian. Mereka hanya berfungsi. Sebagian besar waktu. Kita tidak akan mengutak-atik konfigurasi server web di kelas.

Apache adalah server web di Reclaim dan XAMPP. Ini adalah server web yang paling banyak digunakan.


Bekerja bersama 

Klien dan server bekerja sama untuk menyelesaikan pekerjaan. Klien menampilkan UI. Saat pengguna menginginkan sesuatu, klien mengirimkan permintaan ke server. Server melakukan apa yang perlu dilakukan, dan mengirimkan hasilnya kembali ke klien. 

Misalkan salah satu klien (mungkin milik Anda) dapat mengubah data doggo di server. Begitu data diubah, setiap klien akan mendapatkan data baru saat diminta. W00t!
