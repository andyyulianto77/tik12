Itu ada di server

PHP berjalan di server. Peramban pengguna hanya menerima HTML. Peramban tidak tahu dari mana HTML itu berasal. Peramban hanya menampilkan apa yang diterimanya.

Ketika XAMPP memasang Apache, ia juga memasang interpreter PHP, sebuah program yang dapat menjalankan program PHP. XAMPP menghubungkan interpreter PHP ke Apache.

Jadi:

  1. Peramban mengirimkan URL yang diakhiri dengan PHP ke server web (Apache).
  2. Apache mengambil berkas yang cocok dengan URL dari sistem berkas (disk drive).
  3. Apache memperhatikan .phpekstensi tersebut. Apache mengirimkan file tersebut ke interpreter PHP.
  4. Penerjemah menjalankan kodenya.
  5. Interpreter mengirimkan HTML yang dihasilkan ke Apache.
  6. Apache mengirimkan HTML ke klien (peramban).

Sintaksis

Sintaksnya berbeda dengan VBA. Berikut ini PHP-nya lagi:

  • $cutestDog â€‹â€‹= 'Renata';
  • $peringkat = 10;

  • <p>Anjing yang paling lucu: <?php print $cutestDog; ?></p>
  • <p>Peringkat: <?php print $rating; ?> dari 10.</p>
  • ...

PHP berada di antara <?phpdan ?>. Ini agar penerjemah PHP tahu apa yang harus diperhatikan.

Variabel dimulai dengan $, seperti yang saya katakan. Variabel tidak perlu dideklarasikan, artinya, tidak ada Dimpernyataan, seperti dalam VBA.

Ingatlah bahwa dalam VBA, Anda memberi variabel sebuah tipe, biasanya Singleatau String. PHP memperbolehkan variabel memiliki tipe apa pun yang diinginkan. Tipe bergantung pada bagaimana variabel digunakan, bukan bagaimana ia dideklarasikan. Anda akan melihatnya saat kita membahasnya.

Setiap pernyataan dalam PHP harus diakhiri dengan titik koma (;). Hal ini berlaku untuk banyak bahasa.

Standar penamaan

Kami akan menggunakan beberapa standar pengkodean dalam kursus ini.

Gunakan camel case untuk nama variabel. Semua karakter menggunakan huruf kecil, kecuali jika Anda memiliki nama yang terdiri dari beberapa kata. Jadikan karakter pertama dari kata kedua dan selanjutnya menggunakan huruf besar. Contoh:

  • Jumlah anjing
  • $keasamanrata-rata
  • Pembayaran Bulanan
